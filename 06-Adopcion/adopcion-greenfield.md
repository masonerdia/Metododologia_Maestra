# Adopción · Proyecto nuevo (greenfield)
El caso fácil: arranca con la metodología instalada.
1. Copia los TRANSPORTABLES (`05-Plantillas/`, el adaptador de stack de `03-Gates/Adaptadores/`) al repo nuevo.
2. Instala el índice ([[CLAUDE-base]]) y el pipeline ([[AGENTS-base]]).
3. Define el **nivel de riesgo** ([[rigor-proporcional-al-riesgo]]) y activa los gates que correspondan.
4. Desde el primer día: [[preview-antes-de-construir]], [[mobile-y-desktop-siempre]], [[ci-build-de-produccion]].
La memoria/notas del proyecto nacen vacías en su repo (no se copian de otro).
