# Metodología Maestra

Kit de método **genérico y reutilizable** para construir y mantener software con calidad sostenida — destilado de proyectos reales (referencia: Denti). Para adoptarse en proyectos **nuevos** o **ya en producción**, en **cualquier stack**.

## Cómo usar esta bóveda
- Ábrela como **bóveda en Obsidian**. Punto de entrada: **[[000-MAPA]]**.
- Es también un **repo git** (versiónala a GitHub). El agente [[CURADOR]] propone lecciones nuevas en `_inbox/`; tú las revisas y promueves.

## Dos capas (regla de oro)
1. **BIBLIOTECA** (Obsidian-nativo): principios, gobernanza, rituales, gates, auditorías, mapa. Usa `[[wikilinks]]`, tags, graph. Vive solo en la bóveda.
2. **TRANSPORTABLE** (CommonMark limpio, sin Obsidian-ismos): `05-Plantillas/` y `03-Gates/Adaptadores/`. Se copia tal cual al repo de cada proyecto. Regla: si sale hacia un proyecto, va limpio.

## Principio rector
Aplicar **rigor proporcional al riesgo**: un núcleo mínimo para todo, los gates pesados solo donde hay dinero, datos sensibles o clientes externos. Ver [[rigor-proporcional-al-riesgo]].
